# Labs Experiments

This folder contains early-stage experiments and short technical spikes.

## Purpose
- Validate ideas quickly
- Test tools or architecture decisions
- Capture lessons before production work

## Experiment checklist
- Objective and hypothesis
- Minimal implementation
- Measured result
- Decision: continue, iterate, or discard

## Naming convention
Use date-prefixed or topic-based folder names, for example:
- 2026-04-auth-prototype
- api-rate-limit-spike

Keep experiments lightweight and focused.
